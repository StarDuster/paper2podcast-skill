# Gemini API 503 — 2026-05-12 高峰期 outage 记录

## 时间与现象

日本时间 2026-05-12 23:47 ~ 00:03 期间（UTC 14:47~15:03），Gemini API 的 `gemini-3.1-pro-preview` 模型持续返回 HTTP 503：

```
{
  "error": {
    "code": 503,
    "message": "This model is currently experiencing high demand. Spikes in demand are usually temporary. Please try again later.",
    "status": "UNAVAILABLE"
  }
}
```

## 影响范围

- **script 模型 `gemini-3.1-pro-preview`**：完全不可用，所有请求（context-search、outline-generation、single-stage script generation）都 503。
- **TTS 模型 `gemini-2.5-pro-preview-tts`**：未受影响。
- **flash 模型 `gemini-3-flash-preview`**：未受影响，可用作降级。

## 降级路径

```bash
# 脚本模型改用 flash，TTS 保持 pro
python3 scripts/paper2podcast.py "<input>" --duration 10 --script-model gemini-3-flash-preview
```

## 推测原因

可能是 US 白天时段（~UTC 14-16）Gemini pro 模型的使用高峰。日本夜间常用时间窗口与此重叠。

## 后续建议

- pro 模型的 503 通常是短暂的（几分钟到十几分钟），可以稍后重试。
- 如果用户时间紧迫，直接用 flash 降级，脚本质量虽有下降但可接受。
- `--skip-search` 减少 API 调用次数并不能绕过 503——pro 模型本身不可用，不是 rate limit。
